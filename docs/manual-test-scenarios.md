# 수동 테스트 시나리오

기능별 핵심 동작을 직접 확인하는 체크리스트.
API 호출이 포함되므로 `.env` 파일에 `LAW_API_KEY`와 `OPENAI_API_KEY` 설정 필요.

---

## 시나리오 A — 연관항 검토 (법인세법 제27조의2)

**목적**: ③항 개정 시 ⑤항 [제안]과 ④항 [검토]가 올바르게 생성되는지 확인.

**준비**: 법인세법 검색 → 제27조의2(업무용승용차 관련비용의 손금불산입 등 특례) 선택.

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 개정 요강 입력: "③항 한도금액을 800만원에서 1000만원으로 인상" | — |
| 2 | 부칙 유형: 장래효 → GPT 초안 생성 클릭 | 초안 생성됨 |
| 3 | 연관 항 검토 섹션 확인 | ⑤항 [제안] 2개 표시: "800만원"→"1000만원", "400만원"→"500만원" |
| 4 | ⑤항 체크박스 체크 | 개정안 ⑤항에 원문 전체 + `<u>1000만원</u>`, `<u>500만원</u>` 표시 |
| 5 | ⑤항 체크박스 해제 | 개정안 ⑤항이 `⑤ (현행과같음)` 으로 복귀 |
| 6 | ④항 [검토] 메시지 확인 | "1000만원으로 변경 검토 권고" 수준의 구체적 힌트 포함 |

## 시나리오 B — 병행 법령 검사 (법인세법 ↔ 소득세법)

**목적**: 법인세법 조문 개정 시 소득세법 관련 조문이 탐지되는지 확인.

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 법인세법 조문 선택 후 초안 생성 | — |
| 2 | "병행 법령 검사" 버튼 클릭 | 소득세법 관련 조문 제안 표시 |
| 3 | 제안 조문 expander 열어 이유·내용 확인 | 탐지 이유 + 조문 내용 표시 |
| 4 | 제안 조문 "포함" 체크 → "병행 법령 초안 생성" | 소득세법 조문 초안 생성 완료 |

**주의**: `PARALLEL_LAWS` 매핑에 없는 법령은 검사 대상이 아님 (`config.py` 참조).

## 시나리오 C — HWPX 출력

**목적**: 색상 마크업이 HWPX 파일에 올바르게 반영되는지 확인.

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 1단계 초안 생성 완료 | — |
| 2 | 3단계 탭 이동 → 신구조문대비표 미리보기 확인 | 현행·개정안 두 열 비교 표시 |
| 3 | "HWPX 생성" 버튼 클릭 → 파일 다운로드 | `.hwpx` 파일 생성 |
| 4 | 한/글 또는 호환 프로그램으로 파일 열기 | 현행 삭제 문구: **파란색** 텍스트 |
| 5 | — | 개정안 추가 문구: **빨간색 + 빨간 밑줄** 텍스트 |

---

## 시나리오 D — 인용·준용 규정 파싱 (2단계)

**목적**: 개정안 내 조문 인용이 올바르게 추출·링크되는지 확인.

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 초안 생성 완료 후 2단계 탭 이동 | — |
| 2 | "인용 규정 분석" 확인 | `제X조제Y항` 형태 인용 목록 표시 |
| 3 | 본법 내부 인용 확인 | 조문별 법제처 원문 링크 포함 |
| 4 | 타법 인용 (`「소득세법」 제XX조` 등) 확인 | 별도 섹션에 표시 |
| 5 | "역방향 인용 검사" expander 확인 | 개정 조문을 인용하는 다른 조항 목록 |

---

## 시나리오 E — GPT 응답 파싱 실패 처리

**목적**: GPT가 구분자 없는 응답을 돌려보낼 때 사용자에게 오류가 표시되는지 확인.

이 시나리오는 자동 재현이 어려우므로 코드 레벨에서 확인:
`core/amendment_agent.py`의 `parse_draft_sections()`가 `===SECTION:XXX===` 구분자를
찾지 못하고 키워드 폴백도 실패하면 → 모든 섹션이 빈 문자열.
`ui/stage1_draft.py`에서 이 경우 `st.error("GPT 응답 파싱 실패")` + 원문 expander 표시 확인.
